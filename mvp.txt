## **Role Description**

You have learned all about the foundations of the Web over the past unit. 
Now you will build a static landing page that links to  your group's application.

## **The `Web Foundations` Unit explored the following topics:**

- User Interfaces
- Git for Web Development
- Responsive Design
- Pre-Processors
- JavaScript Fundamentals

For your project this week, you will demonstrate proficiency by working with a group of 
students cross-section to deliver a minimum viable product (MVP) over the course of the sprint.

## **Your primary role User Interface Developer:**

- [ ]  You will build a multi-page marketing website that matches the theme chosen by your group 
       and provides the user with details about the product your team has selected to build.
       For this Marketing site's home page you need to have content and copy that relates 
       to the product that your team is working on.

- [ ]  Your marketing website must include responsive mobile and desktop screen widths. 
       Please see the "Responsiveness" portion of your rubric for more information on this.
       
       Deployed site is responsive to width of browser. All elements are visible at any width of 
       the screen. There is a usable layout at desktop 1000px+ and mobile 500px-. 

- [ ]  The call to action on the website should be a login button that links 
       the user over to the React application that your Front End Architect is building.

- [ ]  Your marketing website must be hosted on a hosting platform like "Netlify" or "Github Pages".

- [ ]  Student has appropriate less files, has used variables and mixins appropriately 
       has nested styles appropriately and has imported their files into index.less in the correct order

- [ ]  Descriptions about the product and about the features it provides would be key. 

- [ ]  For a second page, you could do an about us section that includes images of 
       the members of your team, and links to one another's socials/Github.
